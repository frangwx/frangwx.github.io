RPC是什么

发表于2026-01-15|更新于2026-01-15|java学习日记
|浏览量:

简要说明

什么是RPC，流程和逻辑大致是什么?
是一种抽象的，用于程序间通过网络进行接口调用的框架。使程序员在使用中能像正常本地调用接口一样调用远程服务，其中封装有服务注册与发现、通信协议的选择、数据传输的编码选择以及负载均衡、超时重试、熔断降级等功能。
rpc应当定义定义消息格式（Header + Body）、序列化方式、调用 ID（用于匹配请求/响应）等

服务端：

  1. 服务端向注册中心（比如zookeeper）注册服务端提供的接口（可以通过注解等方式发现端口）；
  2. 启动后监听服务端口，接收到客户端传来的数据后进行通信协议的解码->数据反序列化->调用对应接口模块进行处理->封装结果->序列化->通信协议编码->网络返回给客户端.

客户端:

  1. 链接注册中心发现服务
  2. 业务调用映射的网络接口->动态代理接口->请求信息序列化->协议编码->连接服务端->发送请求->接收请求->协议解码->反序列化->调用完成
  3. 在调用链中,还穿插有负载均衡、超时重试、限流熔断等功能。

时序图

    sequenceDiagram
    participant 服务端
    participant 注册中心
    participant 客户端

    Note over 服务端: 服务启动阶段
    服务端->>注册中心: 1. 注册服务（IP、端口、接口名、元数据等）
    activate 服务端
    注册中心-->>服务端: 2. 注册成功确认
    deactivate 服务端

    Note over 客户端: 客户端启动/调用阶段
    客户端->>注册中心: 3. 查询可用服务列表
    注册中心-->>客户端: 4. 返回服务地址列表

    客户端->>客户端: 5. 动态代理拦截方法调用<br/>（选择服务地址，负载均衡）

    客户端->>客户端: 6. 构造请求：<br/>- 方法名、参数<br/>- 生成 request ID<br/>- 序列化 + 协议编码

    客户端->>服务端: 7. 发送网络请求（TCP/HTTP等）
    activate 客户端
    activate 服务端

    服务端->>服务端: 8. 处理请求：<br/>- 协议解码 → 反序列化<br/>- 根据方法名调用本地实现<br/>- 执行业务逻辑<br/>- 构造响应（含相同 request ID）<br/>- 序列化 + 编码

    服务端-->>客户端: 9. 返回响应
    deactivate 服务端

    客户端->>客户端: 10. 解码响应 → 反序列化<br/>根据 request ID 匹配原始调用

    客户端->>客户端: 11. 返回结果给业务代码
    deactivate 客户端

    Note right of 客户端: （可选）超时重试、熔断、链路追踪等<br/>横切逻辑贯穿调用过程
