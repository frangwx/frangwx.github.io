Java并发基础知识

发表于2026-03-27|更新于2026-03-27|java学习日记
|浏览量:

Java并发编程是Java开发中非常重要的一部分，掌握并发基础知识对于编写高性能、高可用的程序至关重要。

线程基础

1. 线程的创建方式

Java中创建线程主要有三种方式：

方式一：继承Thread类

方式二：实现Runnable接口

方式三：实现Callable接口（可返回结果）

2. 线程的生命周期

线程有以下几种状态：

状态 说明
NEW 新建状态，线程被创建但未启动
RUNNABLE 可运行状态，正在运行或等待CPU调度
BLOCKED 阻塞状态，等待获取锁
WAITING 等待状态，等待其他线程通知
TIMED_WAITING 超时等待状态，等待指定时间后自动唤醒
TERMINATED 终止状态，线程执行完毕

线程同步机制

1. synchronized关键字

synchronized是Java中最基本的同步机制，可以修饰方法或代码块。

修饰实例方法

修饰静态方法

2. ReentrantLock

ReentrantLock是JUC包中提供的可重入锁，比synchronized更灵活。

ReentrantLock vs synchronized

特性 synchronized ReentrantLock
锁获取方式 JVM内置，自动释放 手动获取和释放
是否可中断 不可中断 可中断
是否公平锁 非公平 支持公平/非公平
条件变量 单一条件 多条件变量

3. volatile关键字

volatile用于保证变量的可见性和禁止指令重排序。

volatile的特性：

  • 保证可见性：一个线程修改后，其他线程立即可见
  • 禁止指令重排序：保证有序性
  • 不保证原子性：不能替代锁

JUC并发工具类

1. CountDownLatch

CountDownLatch用于等待多个线程完成。

2. CyclicBarrier

CyclicBarrier用于多个线程互相等待，直到到达某个屏障点。

3. Semaphore

Semaphore用于控制同时访问的线程数量。

2. 四种拒绝策略

策略 说明
AbortPolicy 抛出RejectedExecutionException异常（默认）
CallerRunsPolicy 由调用线程执行任务
DiscardPolicy 直接丢弃任务，不抛异常
DiscardOldestPolicy 丢弃队列中最老的任务，然后重新提交

3. Executors工厂方法（不推荐生产使用）

注意：Executors创建的线程池在阿里巴巴Java开发规范中不推荐使用，因为FixedThreadPool和SingleThreadExecutor允许的请求队列长度为Integer.MAX_VALUE，可能会堆积大量请求导致OOM；CachedThreadPool允许的创建线程数量为Integer.MAX_VALUE，可能会创建大量线程导致OOM。

注意事项

  1. 避免死锁：确保锁的获取和释放顺序一致，尽量使用tryLock设置超时时间

  2. 正确关闭线程池：使用shutdown()或shutdownNow()关闭线程池

  3. 合理设置线程池大小：

    • CPU密集型任务：线程数 = CPU核心数 + 1
    • IO密集型任务：线程数 = CPU核心数 * 2

  4. 使用ThreadLocal注意内存泄漏：使用完毕后调用remove()方法

  5. 优先使用并发集合：避免使用Collections.synchronizedXXX，优先使用ConcurrentHashMap等并发集合
